Project Core Notes

Core will contain:
Business rules
Interfaces
Domain Models
Validation rules

Domain Model design

ProjectRequest - what the user wants to generate
GeneratedProject - result of the generation
ProjectTemplate - Project type template information
InputValidation - holds validation outcomes


Domain definitions:

ProjectRequest:

- ProjectName (string) : the name of the project
- AuthorName (string, optional) : who is generating it (future addition maybe)
- ProjectType (enum) : BasicFrontend, ReactFrontend (future V2)
- IncludeReadme (bool) : whether to generate README
- PageConfig (PageConfiguration) : Holds single/multi page info and number of sections
- Options (List) : Features, Libraries, add-ons

GeneratedProject:

- RootPath (string) : logical root folder
- Files (list of strings) : file paths relative to root
- Warnings (list of string) : any soft warnings from generation

ProjectTemplate:

- Name (string) : e.g., "Basic Frontend"
- RequiredFolders (list of string) : e.g., ["css", "js"]
- RequiredFiles (list of string) : e.g., ["index.html", "base.css"]
- Options (List of strings) : e.g., ["bootstrap", "google fonts",...]
- DefaultPages (int) : Single or Multi Page

InputValidation:

- IsValid (bool)
- Errors (list of string) : e.g., invalid characters, length exceeded
- Warnings (list of string) : e.g., name will be truncated if too long in a future version
